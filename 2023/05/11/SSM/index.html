SSM框架理解

Posted on 2023-05-11 Edited on 2023-08-05

1-SSM框架理解

一、entity层

同类： model层 = entity层 = domain层
作用： 用于存放我们的实体类，与数据库中的属性值基本保持一致。
本项目中：在我们的网页中，video就是一个entity，存储了video的信息，包括ID号，名称等，并且把数据序列化等操作；User也是类似，里面定义了User的id，username，password，使用了Getter and Setter, 还有to string把这些参数转化成string。

二、mapper层

同类： mapper层 = dao层
作用：现在用mybatis逆向工程生成的mapper层，其实就是dao层。对数据库进行数据持久化操作，他的方法语句是直接针对数据库操作的。mapper层是直接跟数据库打交道的，他也是个接口，只有方法名字，具体实现在mapper.xml文件里，service是供我们使用的方法。
本项目中：mapper层针对User的username，password进行了put操作，把数据写入了数据库中。此外，在此mapper层还写了login的函数，使用userMap.get函数，把username和password和之前put进数据库的数据进行函数操作，比如查看是否密码一致，如果一致，就return user。

三、service层

同类： 只有一个 service层
作用： service层是针对我们controller，也就是针对我们使用者。service的impl是把mapper和service进行整合的文件存放业务逻辑处理，也是一些关于数据库处理的操作，但不是直接和数据库打交道，他有接口还有接口的实现方法，在接口的实现方法中需要导入mapper层。
本项目中：UserService里User user = userMapper.login(username,password)就是把mapper和service进行整合，之前mapper层写了login函数，可以把数据库的user给return出来，这里使用这个函数，查看输入的password是否和数据库的password一致，如果一致user就被给传出来，到service层，然后使用了String token= UUID.randomUUID().toString()，会生成一个unique的UUID的string，这里还有函数系统打印token，在用户侧可以看到打印的结果。

四、controller层

同类： controller层 = web 层
作用： 控制器，导入service层，因为service中的方法是我们使用到的，controller通过接收前端传过来的参数进行业务操作，再将处理结果返回到前端。
本项目中：UserController中，String token = userService.login(user.getUsername(), user.getPassword())，token就是使用到了userService的定义的String login(String username, String password)，
然后在controller层通过return token != null ? JsonData.buildSuccess(token, “correct username and password”) : JsonData.buildError(“wrong username and password”);前端通过post把username和password参数传入controller层，controller处理了数据之后，把token的结果返回到前端，在前端可以看到Json的数据，buildSuccess和buildError的函数决定了code,data,msg。但是这里的设计是msg是从controller层传过来的，code和data是JsonData定义的。

五、dto层

同类：只有一个dto层
作用：DTO(数据传输对象层)，该层负责屏蔽后端的实体层，将UI层需要的数据进行重新的定义和封装，在实际的业务场景下，后端实现或存储的数据远比用户需要的数据要庞大和复杂，所以前端需要的数据相对来说要么是组合的，要么是抽取的，不是完整的，因为我们在设计数据存储格式上都会有一些额外的设计和考虑。前端的UI层，只是知道DTO的存在，同时前端需要的数据都在一个DTO中，这样，每次调用服务层的时候，只需要调用一次就可以完成所有的业务逻辑操作，而不是原来的直接调用业务逻辑层那样的，需要调用多次，对于分布式场景下，减少服务调用的次数，尤其重要
